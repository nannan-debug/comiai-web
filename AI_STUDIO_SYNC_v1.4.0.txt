ComiAI v1.4.0 相对 v1.3.0 的增量修改说明

这份文档用于发给 AI Studio，帮助它理解当前版本相对 v1.3.0 新增了哪些页面和交互修改。

版本信息
- 产品名：ComiAI
- 仓库名：comiai-web
- 当前版本：v1.4.0
- 对比版本：v1.3.0

本次主要涉及文件
1. src/App.tsx
2. src/StoryboardManagement.tsx
3. src/StoryboardProduction.tsx
4. src/VideoPreview.tsx
5. package.json
6. package-lock.json
7. README.md

一、整体流程新增
1. 第 3 步“分镜制作”新增“查看视频预览”入口。
2. 第 4 步“视频预览”支持非线性进入，不要求严格按线性流程走完。
3. 当用户从视频预览页切回前面步骤时，如果预览里有未应用的改动，会给出“已自动保存为草稿”的轻提示。

二、App.tsx 的主流程调整
1. 生产流程顶部 step 切换改成统一通过 handleProductionStepChange 处理。
2. 新增 previewDraftState 和 previewNotice，用于管理视频预览页的草稿状态提示。
3. 从第 3 步进入第 4 步时，不再只是手动切 step，而是支持由“查看视频预览”按钮直接打开。
4. VideoPreview 现在会把草稿状态回传给 App，用于跨步骤切换时提示。

三、分镜管理页（StoryboardManagement.tsx）新增和加强
1. 提示词编辑框升级
- 不再只是普通纯文本输入。
- 已绑定的角色 / 场景 / 道具会在 prompt 里以彩色标签形式高亮展示。
- 用户能明确看到某个绑定资产在提示词里被用在了哪里。

2. @ 引用资产能力
- 当用户在 prompt 里输入 @ 时，会弹出当前分镜已绑定资产列表。
- 选择后会插入 @{资产名} 形式的引用。
- 插入后在 prompt 中显示为彩色 tag，而不是普通文字。

3. 绑定资产与 prompt 联动
- 分镜卡的绑定资产区新增 +角色 / +场景 / +道具 按钮。
- 新增绑定资产后，会同步加入当前分镜的资产列表。
- 同时会自动把这个资产追加进 prompt，形成可见引用。

4. 已生成素材的 hover 操作统一
- 在已生成好的图片区或视频区上，鼠标悬浮会出现 3 个按钮：
  - 预览效果
  - 重新生成
  - 精修图片 / 精修视频
- 预览效果会打开浮窗：
  - 图片显示大图预览
  - 视频显示视频预览态和时长

5. 图生 / 直出切换逻辑进一步明确
- 图生视频工作流里：
  - 点“图生”时，显示“图片 + 视频”两个内容区
  - 点“直出”时，只显示“视频”一个内容区
- 不再只是 tab 颜色变化，而是内容结构也会跟着切换。

6. tab 视觉统一
- 分镜管理页每个分镜卡顶部的“图生 / 直出”切换样式已经统一成同一套胶囊 tab。

四、第 3 步分镜制作页（StoryboardProduction.tsx）新增
1. 新增“查看视频预览”按钮。
2. 点击后可直接跳转到第 4 步视频预览页。
3. 这让第 3 步和第 4 步变成非线性关系，用户可以随时打开预览查看效果。

五、第 4 步视频预览页（VideoPreview.tsx）新增和调整
1. 草稿状态改成轻量机制
- 视频预览页内部仍保留草稿概念。
- 用户在预览页的调整会自动保存为草稿。
- 草稿状态会回传给 App，用于离开页面时提示。

2. 去掉顶部大块“预览草稿”说明区域
- 之前大面积的草稿提示区已移除。
- 页面顶部恢复更干净的预览工作区，不再被一整块说明卡占据。

3. 顶部工具栏保留轻量输入
- 当前预览调整说明被收成更轻量的输入位。
- “应用到当前版本”按钮仍然保留在工具区里。

4. 左右边栏支持收起 / 展开
- 左侧“剧集资产”面板现在可以收起和展开。
- 右侧“审阅反馈”面板现在也可以收起和展开。
- 收起后中间播放器区域会变宽，页面更聚焦于视频预览。

六、版本信息修正
1. package.json 版本号升级为 1.4.0。
2. package-lock.json 顶层 name/version 同步修正为 comiai-web / 1.4.0。
3. README 当前版本号更新为 v1.4.0。

如果要让 AI Studio 同步这一版，建议优先替换以下文件：

2. src/StoryboardManagement.tsx
3. src/StoryboardProduction.tsx
4. src/VideoPreview.tsx

如果还要同步版本信息，再一起替换：
