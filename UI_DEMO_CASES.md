# ComiAI UI Demo 案例（基于 `UI_GUIDELINES.md`）

> 用途：给产品/设计/前端一起评审“样式是否统一、文案是否合理、交互是否可落地”。

---

## Demo 1：按钮禁用态 + Hover 原因提示

### 场景
- 页面：`剧本与设定` → `拆分镜预览`
- 按钮：`下一步：制作分镜表`
- 条件：当前方案下“已采用分镜数 = 0”

### UI 展示
- 按钮置灰（背景 `#dddddd`，文字 `#8e9194`）
- 鼠标 hover 按钮显示黑色提示气泡

### Tooltip 文案
- `暂无采用分镜，不可点`

### 验收标准
- 没 hover 时不打扰用户
- hover 时可解释“为什么不能点”

## Demo 2：勾选 vs 采用（交互分离）

### 场景
- 页面：`剧本与设定` → `拆分镜预览` 列表

### 交互规则
- 勾选框：只表示“选中”。
- `采用`按钮：才表示“已采用”。

### UI 状态示例
- A 行：已勾选，未采用 → 按钮显示 `采用`
- B 行：已勾选，已采用 → 按钮显示 `已采用`
- C 行：未勾选 → `采用`按钮置灰，hover 提示 `请先勾选该分镜`

### 顶部统计
- `已选X个`：显示勾选数
- 下一步可点击条件：显示采用数 > 0

## Demo 4：新建导出任务（二级弹窗）

### 标题
- `新建导出任务`

### 字段
- 标签：`导出地址`
- 辅助文案：`请粘贴本地剪映草稿地址`
- 输入框 placeholder：`请输入剪映草稿地址`

- `在剪映的全局设置中可查看剪映的草稿位置，请正确粘贴内容，否则会保存失败。`

### 底部按钮
- `取消`（次按钮）
- `确认`（主按钮）

## Demo 6：分镜号双击编辑

### 场景
- 列表头部展示：`1. 分镜1-1-1`

### 规则
- 左侧 `1.` 为真实序号（不可编辑）
- 右侧 `分镜1-1-1` 可双击进入输入框编辑
- 失焦 / Enter 保存，Esc 取消

### 验收文案
- hover 标注：`双击可编辑分镜号`

---

## Demo 7：剧本文本编辑双态

### 场景
- 左侧 `本集剧本` 区域

### 状态
- 默认只读：按钮文案 `手动编辑`
- 点击后进入编辑：按钮文案变 `保存编辑`
- 再点一次保存并退出只读

### 要求
- 去掉 `导入剧本` 按钮
- 非编辑态不可输入，避免误改

## Demo 8：字号层级示例（防“大字错误”）

- 页面内标题：`16px`

- 弹窗主标题：`24-28px`

- 表格内容 `20px`
- 弹窗正文 `18px+`
- 普通按钮文字 `16px+`
