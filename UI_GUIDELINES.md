# ComiAI UI 规范（持续迭代版）

## 1. 目标
- 避免“字号过大、层级混乱、样式不一致”等低级问题。
- 让设计、前端、产品对同一页面有统一判断标准。
- 支持持续迭代：每次改动都按同一套规则验收。

---

## 2. 核心原则
- 一致性优先：同类型组件必须同样式、同交互。
- 信息层级清晰：主次分明，不靠“大字”制造重点。
- 默认简约：减少装饰，突出任务操作。
- 状态可解释：禁用必须有原因提示（hover 可见）。

---

## 3. 字号规范（重点）

> 禁止随意使用 `text-[30px+]` 作为常规业务文本。

### 3.1 基础字号
- `12px`：辅助说明、标签注释、次级提示。
- `14px`：正文默认、表格内容、输入内容。
- `16px`：小标题、重点正文。
- `18px-20px`：区块标题（页面内）。
- `24px-28px`：弹窗主标题（仅主标题可用）。

### 3.2 强约束
- 弹窗内容区正文：`14px` 默认，最多 `15px`。
- 表格标题与内容：`14px`。
- 按钮文字：`14px`（主按钮可加粗）。
- 禁止出现：
  - 表格内容 `>=16px`
  - 常规按钮 `>=16px`
  - 弹窗正文 `>=16px`

---

## 4. 颜色与状态

### 4.1 主色与中性色
- 主操作色：`#01cd74`
- 主文本：`#1c2329`
- 次文本：`#64748b`（或同级中灰）
- 边框：`#d2d3d4`
- 背景浅灰：`#f8f8f9` / `#f3f4f5`

### 4.2 状态色
- 成功/选中：绿色体系（边框与填充一致）。
- 禁用：背景 `#dddddd`，文字 `#8e9194`。
- 危险操作：红边红字（删除等）。

### 4.3 禁用态规则（必须）
- 禁用按钮必须有原因提示。
- 鼠标 hover 禁用态时展示明确文案，例如：
  - `请先选方案`
  - `暂无采用分镜，不可点`

---

## 5. 组件规范

## 5.1 按钮
- 主按钮：深色/主色填充，白字，统一圆角（`8-12px`）。
- 次按钮：白底 + 灰边 + 深字。
- 危险按钮：浅红边框。
- 同一页面同级按钮高度一致（建议 `36px` 或 `40px`）。

## 5.2 弹窗
- 同类型弹窗使用同一容器模板：
  - 圆角、阴影、关闭按钮位置一致
  - 标题字号一致
  - 底部按钮布局一致（右对齐）
- 二级弹窗（确认/输入）风格必须继承主弹窗。

## 5.3 表格/列表
- 表头与行高保持统一，不做大字号。
- 列标题、内容默认 `14px`。
- 操作列按钮样式与全局按钮规范一致。
- 分页区固定结构：
  - 总条数
  - 每页数量
  - 前后翻页
  - 前往页输入

## 5.4 选择与采用（业务交互）
- 勾选框：仅表示“已选中”。
- 采用按钮：才表示“已采用”。
- 不允许“勾选即采用”的隐式逻辑。

---

## 6. 布局与间距
- 主内容区统一 8px 网格系统：
  - 常用间距：`8 / 12 / 16 / 20 / 24`
- 卡片内间距不小于 `12px`。
- 同一行输入框高度统一。
- 标题与正文的垂直节奏固定，避免忽大忽小。

---

## 7. 文案规范
- 文案短句优先，减少解释性废话。
- 按钮文案使用“动词 + 对象”：
  - `新建导出任务`
  - `刷新列表`
  - `下一步：制作分镜表`
- 禁用原因提示必须是“可执行建议型”文案。

---

## 8. 迭代流程（持续生效）

### 8.1 每次提测前自检（必过）
- 是否存在异常大字（正文/按钮/表格 > 15px）？
- 同类型弹窗是否同模板？
- 禁用态是否有 hover 原因提示？
- 主次按钮、危险按钮样式是否统一？
- 关键流程是否有“选择”和“确认”的明确分离？

### 8.2 Code Review 检查项（必看）
- 是否新增了临时色值、临时字号？
- 是否复用了既有组件而不是重复造型？
- 是否引入了不必要的视觉复杂度？

### 8.3 版本变更记录
- 每次 UI 调整在 release 文档补充：
  - 改了什么
  - 为什么改
  - 影响了哪些页面
  - 是否符合本规范

---

## 9. 禁止项（红线）
- 不允许为了“突出重点”随意放大文字。
- 不允许同一页面出现两套弹窗风格。
- 不允许禁用按钮无原因提示。
- 不允许相同交互在不同页面行为不一致。

---

## 10. 建议落地动作
- 后续统一抽离 `Button / Modal / Tooltip / TablePagination` 基础组件。
- 将本规范映射到 design tokens（颜色、字号、圆角、间距）。
- 在 PR 模板增加 UI 规范勾选项，确保持续执行。
